# Data Update Guide

All catalog data lives in `src/data/` as plain TypeScript objects — no CMS, no database migration
needed for basic content updates. UI components never hardcode vehicle content; they always read
from these files, so edits here propagate everywhere automatically.

## Updating a Vehicle (`src/data/vehicles.ts`)

Every fact that could be wrong carries a `status` field. Only set `status: 'verified'` when you have
a real source (official brochure, confirmed sales price list, etc.):

```ts
startingPrice: {
  value: 285_500_000,
  status: 'verified',
  source: 'Price list internal SAVLUNE, konfirmasi 16 Juli 2026',
  lastUpdated: '2026-07-16',
},
```

Leaving `value: null, status: 'unavailable'` is correct and expected until you have real numbers —
the UI shows "Hubungi sales untuk harga OTR Samarinda terbaru." automatically; it never prints "Rp0"
or a blank field.

Highlights (`highlights: VehicleHighlight[]`) map into the five homepage/detail-page story chapters —
set `chapter` to one of `'presence' | 'comfort' | 'technology' | 'confidence' | 'ownership'`. Leave a
chapter's array empty and the page shows an honest "materi menunggu konfirmasi" note instead of
fabricated copy.

## Adding a Leasing Package (Mode A)

**Manual (small number of packages):** add an entry directly to `financePackages` in
`src/data/finance-packages.ts`. Set `vehicleSlug` and `variantId` once you've confirmed which vehicle
the quotation applies to; leave them `null` with `needsVehicleMapping: true` if not yet confirmed (as
the three Maybank Finance reference rows currently are).

**Bulk (CSV import):** go to `/admin/dashboard` (requires `ADMIN_USERNAME`/`ADMIN_PASSWORD`, see
SETUP.md) and use the CSV import panel. Download the template at
`public/finance-references/leasing-import-template.csv`, fill it in, and upload. Rows are validated
client-side before import and land as `status: 'draft'` in
`src/data/leasing-imports.local.json` (or Supabase once configured) — draft rows are **not** yet wired
into the public calculator. Promote a verified row into `financePackages` manually once checked
against the source document.

CSV columns: `vehicle_slug, variant_id, leasing, scheme, otr, tenor_months, installment_count,
total_first_payment, monthly_installment, region, valid_from, valid_until, verified, notes`.

## Promotions (`src/data/promotions.ts`)

Starts empty on purpose. `/promo` shows an honest "no confirmed promo right now" state with a
WhatsApp CTA rather than a fake promo card. Add an entry only once copy and dates are confirmed.

## Regenerating Placeholder Hero Images

`public/vehicles/<slug>/hero-placeholder.svg` files were generated by a small script (kept for
reference, not part of the build) — replace them with real photography per `ASSET_CHECKLIST.md`
rather than regenerating placeholders.
